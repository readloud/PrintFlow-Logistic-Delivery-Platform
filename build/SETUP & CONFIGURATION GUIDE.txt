## ⚙️ SETUP & CONFIGURATION GUIDE

### A. Prerequisites Installation

```bash
#!/bin/bash
# setup.sh - Complete environment setup

echo "🚀 Starting PRINTFLOW Setup..."

# Update system
sudo apt-get update && sudo apt-get upgrade -y

# Install Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# Install PostgreSQL 15
sudo apt-get install -y postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql

# Install Redis
sudo apt-get install -y redis-server
sudo systemctl start redis-server
sudo systemctl enable redis-server

# Install Nginx
sudo apt-get install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx

# Install Docker & Docker Compose
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# Install Flutter
cd ~
git clone https://github.com/flutter/flutter.git -b stable
echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.bashrc
source ~/.bashrc
flutter doctor --android-licenses
flutter doctor

# Install PM2
sudo npm install -g pm2

# Install AWS CLI
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Install Terraform
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt-get update && sudo apt-get install terraform

echo "✅ Setup complete!"
```

## 🔧 TROUBLESHOOTING GUIDE

### A. Common Issues & Solutions

```yaml
# TROUBLESHOOTING_CHEATSHEET.md

# 1. DATABASE CONNECTION ISSUES
error: "Error: P1001: Can't reach database server"
solution: |
  - Check PostgreSQL service: sudo systemctl status postgresql
  - Verify DATABASE_URL in .env
  - Check firewall: sudo ufw allow 5432
  - Test connection: psql -d postgres -U printflow -h localhost

# 2. REDIS CONNECTION ISSUES
error: "Error: connect ECONNREFUSED 127.0.0.1:6379"
solution: |
  - Start Redis: sudo systemctl start redis-server
  - Check status: redis-cli ping (should return PONG)
  - Check config: sudo nano /etc/redis/redis.conf

# 3. JWT VERIFICATION FAILED
error: "JsonWebTokenError: invalid signature"
solution: |
  - Ensure JWT_SECRET is same across all services
  - Check if secret is at least 32 characters
  - Regenerate secret: openssl rand -base64 32

# 4. COGNITO AUTHENTICATION FAILED
error: "NotAuthorizedException: Incorrect username or password"
solution: |
  - Verify user exists in Cognito User Pool
  - Check if account is confirmed
  - Reset password via admin: aws cognito-idp admin-set-user-password

# 5. SES EMAIL NOT SENDING
error: "Email address is not verified"
solution: |
  - Move from sandbox to production via AWS Console
  - Or add email to verified identities for testing
  - Check sending limits: aws ses get-send-quota

# 6. SOCKET.IO CONNECTION FAILED
error: "WebSocket connection to 'wss://...' failed"
solution: |
  - Check Nginx WebSocket proxy configuration
  - Verify CORS settings in backend
  - Check firewall: sudo ufw allow 3000
  - Test with: wscat -c ws://localhost:3000

# 7. MIDTRANS PAYMENT ERROR
error: "Unauthorized: Invalid server key"
solution: |
  - Check environment: sandbox vs production
  - Verify server key matches environment
  - Check if API is enabled in Midtrans dashboard

# 8. FLUTTER BUILD FAILED
error: "Execution failed for task ':app:compileFlutterBuildRelease'"
solution: |
  - Clean build: flutter clean
  - Get packages: flutter pub get
  - Check Gradle version in android/build.gradle
  - Run: flutter doctor -v

# 9. AWS LAMBDA TIMEOUT
error: "Task timed out after 10.00 seconds"
solution: |
  - Increase Lambda timeout in configuration
  - Optimize code for faster execution
  - Move heavy processing to async queues
  - Use Step Functions for long-running tasks

# 10. SQS MESSAGE NOT PROCESSING
error: "Message not visible in queue"
solution: |
  - Check visibility timeout settings
  - Ensure dead-letter queue is configured
  - Check worker is running: pm2 list
  - Monitor queue: aws sqs get-queue-attributes
```

### B. Health Check Script

```javascript
// scripts/health-check.js
const axios = require('axios');
const { PrismaClient } = require('@prisma/client');
const redis = require('redis');

const prisma = new PrismaClient();

async function healthCheck() {
  console.log('🔍 Running Health Check...\n');
  
  const results = {
    database: false,
    redis: false,
    api: false,
    websocket: false,
    ses: false,
    cognito: false
  };
  
  // Check Database
  try {
    await prisma.$queryRaw`SELECT 1`;
    results.database = true;
    console.log('✅ Database: Connected');
  } catch (error) {
    console.log('❌ Database:', error.message);
  }
  
  // Check Redis
  try {
    const client = redis.createClient({ url: process.env.REDIS_URL });
    await client.connect();
    await client.ping();
    await client.disconnect();
    results.redis = true;
    console.log('✅ Redis: Connected');
  } catch (error) {
    console.log('❌ Redis:', error.message);
  }
  
  // Check API
  try {
    const response = await axios.get('http://localhost:3000/health');
    if (response.status === 200) {
      results.api = true;
      console.log('✅ API: Up');
    }
  } catch (error) {
    console.log('❌ API:', error.message);
  }
  
  // Print Summary
  console.log('\n📊 Summary:');
  const allHealthy = Object.values(results).every(v => v === true);
  console.log(allHealthy ? '✅ All systems operational' : '⚠️ Some services are down');
  
  return results;
}

healthCheck().then(() => process.exit(0));
```
